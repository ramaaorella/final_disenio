## Review inputs

### Design purpose
<dl>
  <dd>
    <p align="justify">
      El próposito de diseño es realizar el suficiente trabajo de diseño para satisfacer los requerimientos del nuevo sistema, guiar el proceso de desarrollo y prepararse para un eventual lanzamiento de la plataforma.
    </p>
    <p align="justify">  
      El diseño parte de una arquitectura nocional inicial (ya implementada a nivel prototipo), que requiere refinarse, luego de entender y desglosar la misma, para lograr una versión mejorada que se ajuste a los objetivos de negocio y demás requisitos, y construir un nuevo incremento del sistema.  
    </p>
    <p align="justify">  
      Desde la organización el propósito de diseño es en favor de obtener principalmente una plataforma robusta, escalable en términos de performance, extensible para nuevas funcionalidades, y que provea una buena experiencia de usuario.  
    </p>
  </dd>
</dl>

### Primary functional requirements
<dl><dd>
  <table>
    <tr>
      <th>ID</th>
      <th>Functional requirement</th>
    </tr>
    <tr>
      <td>UC-1</td>
      <td align="justify">
        El inicio del Portal (interfaz Web) debe cargar toda la información de las bases de datos y mostrarla al operador, y/o al gestor correspondiente.
      </td>
    </tr>
    <tr>
      <td>UC-2</td>
      <td align="justify">
        El ingreso de mensajes (conversaciones) de usuarios a través de un canal, deben ser gestionados, persistidos, y mostrados al operador.
      </td>
    </tr>
    <tr>
      <td>UC-3</td>
      <td align="justify">
        El egreso de mensajes (conversaciones) de operadores, utilizando un canal determinado, deben ser transmitidos al usuario correspondiente.
      </td>
    </tr>
    <tr>
      <td>UC-4</td>
      <td align="justify">
        Gestión mediante dashboards y reportes varios (en modalidad batch).
      </td>
    </tr>
    <tr>
      <td>UC-5</td>
      <td align="justify">
        Integración a futuro con APIs de otros sistemas (por ej., un CRM).
      </td>
    </tr>
  </table>
</dd></dl>

### Constraints

<dl><dd>
  <table>
    <tr>
      <th>ID</th>
      <th>Constrain</th>

      <td>CON-1</td>
      <td align="justify">
        El prototipo está implementado con tecnologías Java y MySQL (mayormente), mediante el framework JHipster para Spring; incluye el uso de colas RabbitMQ.

<dl><dd>
